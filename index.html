NgAndroid

NgAndroid is a compile time MVC framework inspired by AngularJS. NgAndroid sticks to the philosophy that views should be separate from the models and the controls. It allows you to declare view - model bindings and events straight from your xml layout that will interact with the scope/controller.

Model

To start using NgAndroid you are going to need a model. A model is any class with a default constructor that has accessible getters and setters for the fields that are going to interact with the view.

public class Note {
    private long time;
    private int id;
    private String title;
    private String text;


    public long getTime() {
        return time;
    }

    public void setTime(long time) {
        this.time = time;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getText() {
        return text;
    }

    public void setText(String text) {
        this.text = text;
    }
}

Controller

Then you'll need a scope. A scope is the scope of the controller. Everything is defined within and referenced from that scope. A scope can be anything, an Activity, Fragment, or even a separate encapsulated class can be defined as scopes. 

// Declare your scope
@NgScope
public class NoteFragment extends Fragment {
    // declare your models within your scope
    @NgModel
    Note note;

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        // inflate your view with your instance of NgAndroid
        View v = ngAndroid.inflate(this, inflater, R.layout.fragment_note, container, false);
        ...
        return v;
    }

    // A method that will be called on a view event
    String getTimeString(long time){
        return TimeUtils.getTimeString(time);
    }
}

View

Finally, you'll need your view. NgAndroid supports a small scripting language within your xml views that makes the 'magic' happen. There will be more documentation on this script later and this will be updated when that happens.

<ScrollView ...
    xmlns:tools="http://schemas.android.com/tools"
    xmlns:ngAndroid="http://schemas.android.com/apk/res-auto"
    tools:ignore="MissingPrefix">

    <RelativeLayout ...>

        <EditText
            android:id="@+id/et_title"
            ngAndroid:ngModel="note.title"
            .../>

        <TextView
            android:id="@+id/tv_time"
            ngAndroid:ngText="getTimeString(note.time)"
            .../>

        <EditText
            android:id="@+id/et_text"
            ngAndroid:ngModel="note.text"
            .../>
    </RelativeLayout>
</ScrollView>

With the combination of the model,view, and controller you know have a fully interactive user interface. The field 'title' in your Note model will always reflect what is contained in the et_title EditText and vice versa. The TextView tv_time will always be updated with the current time as formatted through the getTimeString(long) method, but it is a one way binding. The field 'time' in your Note model will not be affected by your TextView. Note.text and et_text are also bound two way. Both always reflecting the state of the other.

Usage

In order to use NgAndroid you will need Gradle. Add this to your build.gradle file in your app's module.

buildscript {
    repositories {
        jcenter()
    }
    dependencies {
        classpath 'com.neenbedankt.gradle.plugins:android-apt:1.4'
    }
}

allprojects {
    repositories {
        maven {
            url 'http://oss.sonatype.org/content/repositories/snapshots'
        }
    }
}
apply plugin: 'com.neenbedankt.android-apt'

dependencies {
    compile 'com.github.davityle:ngandroid:1.0.6-SNAPSHOT'
    apt 'com.github.davityle:ng-processor:1.0.6-SNAPSHOT'
}

The allprojects.repositories.maven snapshot url will not be necessary once an official version is released.
